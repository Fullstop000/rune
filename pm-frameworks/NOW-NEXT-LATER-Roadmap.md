# NOW-NEXT-LATER Roadmap (NOW-NEXT-LATER 路线图)

**Category:** Execution (执行落地)

## Overview

A product roadmap framework that replaces date-driven quarterly timelines with three buckets: NOW, NEXT, and LATER. The further out, the more outcomes are described rather than features.

**Origin:** Product management best practices

---

## Three Horizons

| Horizon | Confidence | Description |
|---------|------------|-------------|
| **NOW** | High | Task-level: Currently working on |
| **NEXT** | Medium | Feature-level: Direction confirmed, pending execution |
| **LATER** | Low | Outcome-level: Problems to solve, not features to build |

---

## Key Principle

LATER uses outcome descriptions, not feature descriptions, preserving space for future exploration.

---

## Case Study: Intercom (2017)

**Results after switching from date-driven roadmaps:**
- Quarterly planning time reduced 60%
- Sales/customer "feature delay complaints" significantly reduced
- Several LATER items were never built — considered success, not failure

**Key Quote:** "移除日期之后，对话从『为什么 X 延期了？』变成了『什么是现在最有价值的事？』"

```markdown
# Now-Next-Later Roadmap

Organize product plans into three time horizons to communicate strategy without false precision on dates. Use when stakeholders need roadmap visibility but fixed-date commitments are unreliable or counterproductive.

## Steps

1. **Now** — list items currently in progress or committed for the immediate term (this sprint/cycle).
2. **Next** — list items that are shaped and likely to start after current work finishes (1-3 months out).
3. **Later** — list items that are identified but not yet shaped; directional only (3-6+ months).
4. **Add context** — for each item, note the problem it solves and the target outcome.
5. **Review regularly** — update the roadmap every cycle; items flow from Later → Next → Now as they are shaped and prioritized.

## Output Format

A three-column board (Now / Next / Later) with each item showing: title, one-line problem statement, and target outcome. Include a last-updated date and review cadence.
```
