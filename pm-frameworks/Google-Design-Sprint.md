## Overview

A 5-day structured innovation process developed by Google Ventures (Jake Knapp). Compresses months of planning and development into five focused days.

**Creator:** Jake Knapp, Google Ventures

**Year:** 2010-2012

---

## The Five Days

| Day | Focus | Activities |
|-----|-------|------------|
| **Day 01** | Map/Define | Establish 2-year long-term goal, define Sprint question, draw user journey map |
| **Day 02** | Sketch | Lightning Demos, 4-Step Sketching (Notes → Ideas → Crazy 8s → Detailed solution) |
| **Day 03** | Decide | Art Museum walk, Heat Map voting, Storyboard expansion to 15-frame journey |
| **Day 04** | Prototype | Build realistic prototype using Keynote, Figma, or physical props |
| **Day 05** | Test | Test with 5 target users (60 min each), team watches live, pattern discovery |

---

## Case Study: Slack Enterprise Admin Controls (2016)

**Challenge:** Validate enterprise IT management features before 6+ month development commitment.

**Result:** Sprint identified GDPR compliance settings and audit logs as critical features. 4/5 IT admins asked about GDPR; 3/5 identified audit logs as most important.

**Impact:** Avoided 6 months building wrong direction; enabled Slack Enterprise Grid launch.

## Skill Content

```markdown
# Google Design Sprint (Jake Knapp)

Compress months of work into five days: understand the problem, sketch solutions, decide on one, build a prototype, and test with real users. Use when facing a big product decision and you need fast, validated direction.

## Steps

1. **Monday — Map** — define the challenge, pick a target user and moment, set a sprint goal.
2. **Tuesday — Sketch** — each team member sketches detailed solution ideas individually.
3. **Wednesday — Decide** — vote on the best ideas, pick a winner, create a storyboard for the prototype.
4. **Thursday — Prototype** — build a realistic, clickable prototype (fake it, don't build it).
5. **Friday — Test** — run 1-on-1 usability sessions with 5 target users; capture patterns and insights.

## Output Format

A sprint summary with: the sprint goal, the winning solution storyboard, the prototype link, user test findings organized by pattern, and a clear go/pivot/kill recommendation.
```
