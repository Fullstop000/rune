# Threshold of Viable Ideas (创意可行性门槛)

**Category:** Validation (验证测试)

**Framework ID:** #45

**Source:** https://pmframe.works/framework-threshold-viable-ideas

## Overview

Sets minimum viable standards for idea screening, systematically filtering low-quality ideas to focus resources on truly promising directions.

---

## Process

1. **Define criteria** — Minimum viability standards
2. **Screen ideas** — Against each criterion
3. **Score** — Rate viability
4. **Filter** — Remove below threshold
5. **Advance** — Top ideas move forward

---

## Criteria Examples

| Criteria | Description |
|----------|-------------|
| Market size | Potential reach |
| Technical feasibility | Can we build it? |
| Strategic fit | Does it align with goals? |
| Resource requirements | What do we need? |
| Risk level | How risky is it? |

## Skill Content

```markdown
# Threshold of Viable Ideas

Screen ideas against minimum viability criteria to filter out low-potential concepts early.

## Steps

1. Define 3-5 **minimum viability criteria** -- non-negotiable requirements for any idea to advance (e.g., addresses a real user pain, technically feasible, aligns with strategy).
2. Evaluate each idea against each criterion -- mark as Pass or Fail.
3. Filter out ideas that fail any criterion -- these are below the threshold.
4. Score remaining ideas on a simple 1-3 scale for potential impact and feasibility.
5. Rank surviving ideas by composite score and allocate resources to the top candidates.

## Output Format

A filtering table with rows for each idea and columns for each criterion (Pass/Fail). Include impact and feasibility scores for surviving ideas and a ranked shortlist.
```
