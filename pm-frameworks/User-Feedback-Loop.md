# User Feedback Loop

**Category:** Execution (执行落地)

## Description

Complete closed loop: collect feedback → classify → schedule → notify users, making users feel heard and improving retention.

## The Loop

1. **Collect** - Gather feedback from multiple channels
2. **Classify** - Categorize and prioritize
3. **Schedule** - Plan when to address
4. **Notify** - Update users on progress
5. **Iterate** - Continuously improve

## Benefits

- Users feel heard
- Builds trust and loyalty
- Improves product-market fit
- Creates engagement

## When to Use

- Customer success
- Product development
- Community building
- Retention strategies

```markdown
# 用户反馈闭环 Skill

## 概述

用户反馈闭环是一套系统化的机制，确保用户提交反馈后能够获得回应，从而建立信任并提升产品留存率。

## 核心四环节

### 1. 收集反馈
建立统一的反馈入口，整合所有渠道（应用内、客服、App Store、社交媒体）到单一数据库。

### 2. 分类归因
为反馈打上结构化标签：类型（Bug/需求/体验/内容）和影响范围，便于后续处理和聚合。

### 3. 排期处理
将高优先级反馈正式纳入产品 Roadmap，使用影响×频率×战略相关度评分模型决定优先级。

### 4. 告知用户
这是最关键的环节。区分个性化回复（直接联系提交者）和群体公告（更新日志），功能上线后24小时内通知用户。

## 关键执行要点

- 指定唯一 DRI（直接责任人），避免"大家都负责"
- 每周固定时间处理反馈分类，不依赖临时处理
- 回复模板+个性化修改，增加人情味
- 建立"功能上线通知"触发器，自动通知相关用户
- "不做"也要告知，并给出理由
- 追踪"闭环率"作为团队 KPI

## 常见陷阱

1. 把"收集"当作"处理"——收集工具≠闭环
2. 只回复正面反馈，忽视负面反馈——负面反馈用户是还在乎产品的人
3. 等功能上线才联系——中间状态的主动沟通同样重要

## 量化价值

当用户知道自己提交的Bug已被修复时，其90天留存率比未告知用户高出37%（Intercom研究数据）。
```
