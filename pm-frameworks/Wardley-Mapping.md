# Wardley Mapping (Wardley 地图)

**Category:** Systems Thinking (系统思维)

**Framework ID:** #73

**Source:** https://pmframe.works/framework-wardley-mapping

---

## Overview

Maps all components in value chain by "evolution stage" (Genesis → Custom → Product → Commodity) for differentiated strategy.

**Creator:** Simon Wardley

**Year:** ~2005 at Fotango (Yahoo UK subsidiary)

---

## Evolution Stages

| Stage | Description |
|-------|-------------|
| **Genesis** | Novel, emerging |
| **Custom Built** | Tailored to need |
| **Product** | Mass produced |
| **Commodity** | Utility, everyone has it |

---

## Map Structure

- **Y-axis:** Value chain (user need at bottom)
- **X-axis:** Evolution stage (Genesis to Commodity)

---

## Case Study: UK Government Digital Service (GDS) 2011-2014

**Problem:** Commoditized infrastructure was being custom-developed across departments

**Action:** Built GOV.UK unified platform, "Cloud First" policy

**Result:** Billions saved; 80%+ user satisfaction; adopted by 17+ governments

# Wardley Mapping

Visualize your value chain by plotting components from user need to underlying infrastructure, positioned by their evolutionary stage.

## Steps

1. Start with the **user need** at the top of the map
2. List all **components** needed to serve that need (capabilities, services, data, infrastructure)
3. Map **dependencies** -- draw the chain from user-visible to invisible components
4. Position each component on the **evolution axis**: Genesis -> Custom -> Product -> Commodity
5. Identify **strategic moves** -- build custom in Genesis, buy commodity, watch Product for disruption
6. Spot components ripe for **inertia** (resisting evolution) or **climatic patterns** (inevitable shifts)

## Output Format

**Anchor:** [User need]

| Component | Evolution Stage | Build/Buy/Partner | Strategic Note |
|-----------|----------------|-------------------|---------------|
| [Name] | [Genesis/Custom/Product/Commodity] | [Decision] | [Note] |

**Key moves:**
1. [Component] -- [Action and rationale]
2. [Component] -- [Action and rationale]

**Watch list:** [Components about to shift stage]
