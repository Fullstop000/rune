# Shape Up (Shape Up)

**Category:** Execution (执行落地)

**Framework ID:** #53

**Source:** https://pmframe.works/framework-shape-up

---

## Overview

Basecamp's product development methodology featuring 6-week fixed cycles, fixed budget with floating scope, and problem shaping rather than task lists—eliminating endless backlogs.

**Creator:** Ryan Singer (Basecamp)

**Year:** 2019 (published as free ebook)

---

## Core Principle

Shape Up inverts Scrum's logic: "Time is fixed, scope is floating." Instead of estimating work you don't fully understand, teams find the best solution within a fixed time budget.

---

## The Shaped Work Process

| Phase | Description |
|-------|-------------|
| **Shaping** | PM/designer defines problem boundaries, writes Pitch with problem, appetite, rough solution, out-of-scope items |
| **Betting Table** | ~2 hour decision meeting where management reviews Pitches and commits to work |
| **Building** | 6 weeks with small team (1 designer + 1-2 engineers), full autonomy, no daily standups, Hill Chart tracking |
| **Cool-down** | 2 weeks for bug fixes, tech debt, exploration |

---

## Key Concepts

| Concept | Description |
|---------|-------------|
| **Appetite** | "How much time is this worth?" decided before solutions |
| **Hill Chart** | Progress visualization where uphill = exploring unknown, downhill = certain execution |
| **Pitch** | Rough solution outline with problem, appetite, rough solution, and explicit exclusions |

---

## Common Pitfalls

1. Writing Spec instead of Pitch
2. Skipping Shaping phase
3. Keeping a backlog
4. Not trusting the team during Building
5. Forcing Hill Chart as mandatory
6. Using Shape Up for maintenance work

---

## Case Study: Basecamp x HEY Calendar Feature (2020)

- **Appetite:** 2 weeks
- **Problem:** Users couldn't browse days with email exchanges
- **Solution:** Flip timeline to calendar format
- **Explicit exclusion:** No recurring events in v1
- **Result:** Completed in 2 weeks what competitors took months

## Skill Download

**File:** `skills/shape-up.md`

**Skill Content:**

```markdown
# Shape Up (Basecamp)

Define work at the right level of abstraction, bet on shaped pitches, and give teams fixed time with variable scope. Use when you want to avoid both micromanagement and unbounded projects.

## Steps

1. **Shaping** — define the problem, set the appetite (how much time it deserves), sketch a solution at the right abstraction level, and identify rabbit holes.
2. **Betting Table** — leadership reviews shaped pitches and bets on which to build in the next 6-week cycle.
3. **Building** — a small team (1 designer + 1-2 developers) gets the full cycle to ship; they decide scope trade-offs.
4. **Cooldown** — 2-week buffer after each cycle for bug fixes, exploration, and prep for next shaping round.
5. **No backlogs** — pitches that are not bet on are discarded; if still important, they can be re-shaped and re-pitched later.

## Output Format

A shaped pitch document containing: problem statement, appetite (time budget), solution sketch with fat-marker wireframes, rabbit holes and no-go boundaries, and a go/no-go betting decision.
```
