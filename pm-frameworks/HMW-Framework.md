# HMW Framework (How Might We)

**Category:** Definition (问题定义)

**Framework ID:** #13

**Source:** https://pmframe.works/framework-hmw

---

## Overview

HMW ("How Might We") transforms user research insights into design opportunities, triggering creative ideation. The art is balance - too narrow becomes a solution, too broad loses focus.

---

## Methodology - Width Spectrum

| Level | Example |
|-------|---------|
| **Too narrow** (already a solution) | "How might we reduce registration from 7 steps to 3?" |
| **Just right** | "How might we let new users experience core value in 5 minutes?" |
| **Too broad** | "How might we make the product better?" |

---

## Insight to HMW Conversion Flow

1. User Insight (from POV)
2. Main HMW question
3. Multiple angle HMWs from same insight (child's perspective / friction reduction / emotional angle)

---

## Common Starters

- Help...
- Eliminate...
- Reduce friction of...
- Make... more fun
- Leverage the power of...

---

## Case Study: IDEO x Bank of America "Keep the Change" (2005)

- Insight: Mothers的习惯四舍五入when budgeting
- HMW: "How might we help people save naturally without thinking?"
- Result: Every card purchase rounded up,差额转入储蓄. 2.5M registered users in 1 year, $1B transferred to savings.

---

## Five Key Tips

1. Start from POV transformation to ensure HMW addresses real user problems
2. Write 5-10 HMWs per insight for sufficient creative space
3. Use HMW voting for team alignment
4. HMW is not the final answer - don't solidify too early
5. A good HMW makes everyone want to start sketching immediately

## Skill Content

```markdown

description: Use when reframing problems as actionable opportunity questions
---

# How Might We

Reframe problem statements into open-ended "How Might We...?" questions that are broad enough to invite creative solutions but narrow enough to provide useful constraints.

## Steps

1. **Start with an insight or pain point** — Take a specific user need, frustration, or observation from research.
2. **Write the HMW question** — Phrase it as "How might we [verb] [desired outcome] for [user]?" Avoid embedding a solution in the question.
3. **Check scope** — If the question is too broad (could mean anything), narrow it. If too narrow (implies one answer), broaden it.
4. **Generate variations** — Rewrite the same insight as 3-5 different HMW questions to explore different angles.
5. **Select and prioritize** — Vote or discuss to pick the 1-2 HMW questions that best frame the opportunity for ideation.

## Output Format

A list of 3-5 HMW questions per insight, with the selected question(s) highlighted and a brief rationale for why they best frame the opportunity.
```
