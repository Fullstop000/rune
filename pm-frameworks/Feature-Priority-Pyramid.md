# Feature Priority Pyramid (特性优先级金字塔)

**Category:** Execution (执行落地)

**Framework ID:** #58

**Source:** https://pmframe.works/framework-feature-priority-pyramid

---

## Overview

将产品功能按层次分为五个优先级层级（功能性基础→可靠性→易用性→体验优化→差异化创新），确保产品先满足基础需求，再追求体验卓越。

---

## Five-Tier Structure

| Tier | Name | Description | Acceptance Criteria |
|------|------|-------------|-------------------|
| **L1** | 功能性基础 | 产品的核心功能能够正常工作 | 能用 |
| **L2** | 可靠性 | 系统稳定不崩溃、数据不丢失、性能可接受 | 信任基础 |
| **L3** | 易用性 | 清晰的导航、低认知负担、流畅任务完成路径 | 无摩擦 |
| **L4** | 体验优化 | 视觉美感、交互动效、情感化设计、个性化推荐 | 愉悦感知 |
| **L5** | 差异化创新 | 独特功能创新、品类定义特性、竞争壁垒建设 | 竞争优势 |

---

## Core Insight

用户对产品的需求是分层的：首先要求功能能用（第一层），其次要求稳定不出错（第二层），再次要求用起来顺手（第三层），然后才期待体验愉悦（第四层），最后才在意差异化独特价值（第五层）。

---

## Case Study: Notion

2018-2019年，凭借极高的设计自由度（L4）和创新的块状编辑（L5）积累了大批用户，但同期L2可靠性严重不足。2019-2021年Notion投入大量工程资源重写数据同步层，大型workspace打开速度提升3-5倍。L2夯实后，2021年推出的Notion AI站在稳定基础上获得极高留存率，直接打开企业市场。

## Skill Content

```markdown
# Feature Priority Pyramid

Organize features into three layers — Foundation, Differentiators, Delighters — to ensure basics are solid before investing in delight.

## Steps

1. List all candidate features and requirements.
2. Classify each as **Foundation** (must-have, table stakes — users leave without these), **Differentiator** (competitive advantage — why users choose you), or **Delighter** (unexpected value — why users love and recommend you).
3. Validate that all Foundation items are fully addressed before allocating resources to upper layers.
4. Within each layer, rank by effort-to-impact ratio.
5. Build the roadmap bottom-up: Foundation first, then Differentiators, then Delighters.

## Output Format

Three-tier feature list with classification rationale, priority rank within each tier, and a phased roadmap showing build order.
```
