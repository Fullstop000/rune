## Overview

Seven-step PM interview/design framework: Comprehend → Identify → Report → Cut through → List → Evaluate → Summarize.

---

## The Seven Steps

| Step | Description |
|------|-------------|
| **C - Comprehend** | Understand the situation |
| **I - Identify** | Identify the user |
| **R - Report** | Report user needs |
| **C - Cut through** | Prioritize critical issues |
| **L - List** | List solutions |
| **E - Evaluate** | Assess trade-offs |
| **S - Summarize** | Present recommendations |

# CIRCLES Method (Lewis Lin)

A seven-step framework for answering product design questions comprehensively and systematically.

## Steps

1. **Comprehend** the situation -- clarify the question, the company, and constraints
2. **Identify** the customer -- define 2-3 user segments and select one to focus on
3. **Report** customer needs -- list the top pain points or goals for the chosen segment
4. **Cut** through prioritization -- pick the #1 need to solve using impact vs effort
5. **List** solutions -- brainstorm 3-5 solution ideas for the chosen need
6. **Evaluate** tradeoffs -- compare solutions on feasibility, desirability, and viability
7. **Summarize** your recommendation -- state the winning solution and why

## Output Format

- **Situation:** [Context and constraints]
- **Target User:** [Chosen segment + persona]
- **Top Need:** [Primary pain point]
- **Solutions Considered:** [3-5 options with pros/cons]
- **Recommendation:** [Chosen solution + rationale]
