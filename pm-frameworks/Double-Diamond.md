# Double Diamond (双钻模型)

**Category:** Definition (问题定义)

**Framework ID:** #32

**Source:** https://pmframe.works/framework-double-diamond

---

## Overview

由英国设计委员会于 2005 年提出，双钻模型将设计过程拆解为两次「发散→收敛」循环：第一颗钻石找到正确的问题，第二颗钻石找到正确的方案。跳过第一颗钻石，往往意味着把精力押注在了错误的方向上。

**Originator:** 英国设计委员会 (British Design Council)

**Year:** 2004年正式发布，2005年广泛传播

---

## Four Phases

| Phase | Name | Direction | Core Task |
|-------|------|----------|-----------|
| 01 | **Discover 发现** | 发散 | 广泛探索问题域，收集用户研究数据 |
| 02 | **Define 定义** | 收敛 | 综合洞察，聚焦核心问题，产出 HMW 问题句 |
| 03 | **Develop 发展** | 发散 | 针对已定义问题，发散生成大量创意方案 |
| 04 | **Deliver 交付** | 收敛 | 收敛至最优方案，迭代精炼，最终上线 |

---

## Typical Outputs

| Phase | Outputs |
|-------|---------|
| **Discover** | 用户访谈记录、观察日志、调研报告、竞品分析、用户旅程草图 |
| **Define** | 亲和图、洞察卡、问题陈述 (POV)、HMW 问题句、设计简报 |
| **Develop** | 概念草图、故事板、低保真原型、用户测试报告、方案对比矩阵 |
| **Deliver** | 高保真原型、上线方案、A/B 测试结果、发布计划、效果度量报告 |

---

## Case Study: NHS × IDEO 急诊室患者体验改造 (2013)

- **发现**: 研究员在 6 家 NHS 医院进行数周沉浸式调研，访谈 200+ 名患者及家属，发现关键现象：患者最频繁的行为不是问「还要等多久」，而是不断询问「他们知道我在这里吗」
- **定义**: 反直觉洞察——患者焦虑的根本原因不是等待时间过长，而是不知道自己在等什么
- **发展**: 生成 60+ 个概念方案，筛选 8 个进行原型测试，低成本「可见性」方案胜出
- **交付**: 部署患者旅程看板 + 护士主动沟通脚本 → 患者满意度提升 28%

---

## Five Usage Principles

1. 第一颗钻石不能跳过
2. Define 阶段的产出是 HMW 问题，不是解决方案
3. Discover 阶段要有真实的用户接触
4. 两次发散都要真正「发散」（至少产出 20 个方案才开始筛选）
5. 用已有工具（用户旅程图、亲和图、服务蓝图等）喂养第一颗钻石

**Skill Content:**

```markdown
# Double Diamond

A design process framework with four phases — Discover, Define, Develop, Deliver — that alternate between divergent thinking (exploring widely) and convergent thinking (narrowing down).

## Steps

1. **Discover (diverge)** — Research broadly. Explore the problem space through user research, data analysis, and stakeholder interviews without filtering.
2. **Define (converge)** — Synthesize findings into a clear, focused problem statement. Narrow from many observations to the one problem worth solving.
3. **Develop (diverge)** — Generate a wide range of possible solutions through brainstorming, sketching, and prototyping. Explore multiple directions.
4. **Deliver (converge)** — Select the best solution, refine it, test it, and ship it. Focus on execution and quality.
5. **Iterate** — Use feedback from Deliver to loop back into Discover or Develop as needed.

## Output Format

A phase-by-phase summary showing: key activities performed, artifacts produced, decisions made at each convergence point, and the rationale for narrowing.
```
