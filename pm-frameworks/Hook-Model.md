# Hook Model (上钩模型)

**Category:** Growth (增长策略)

**Framework ID:** #59

**Source:** https://pmframe.works/framework-hook-model

---

## Overview

Nir Eyal 在《上钩》(Hooked) 中提出的四步循环模型：**触发→行动→奖励→投入**。帮助产品设计出让用户自发养成使用习惯的内在机制，摆脱对昂贵外部营销的依赖。

**Creator:** Nir Eyal

**Year:** 2014

---

## Core Structure

### 1. Trigger (触发)

启动行为的开关，分为：
- **外部触发**：推送通知、广告、朋友推荐
- **内部触发**：无聊、焦虑、孤独等情绪状态
- **目标**：外部→内部转化

### 2. Action (行动)

触发后最简单的目标行为。基于 B.J. Fogg 的 B = M × A × T 模型：
- **动机** (Motivation)
- **能力** (Ability)
- **触发** (Trigger)

关键：降低行动摩擦，用最小必要步骤。

### 3. Variable Reward (可变奖励)

"可变"是关键——不可预测性创造渴望：
- **社会奖励**：被点赞/被认可
- **猎物奖励**：发现好内容/信息
- **自我奖励**：完成任务的成就感

### 4. Investment (投入)

用户为未来收益存入"价值"：
- 数据与内容
- 社交关系图谱
- 转换成本

投入连接两次循环，每次投入都加载下一次触发的弹药。

---

##循环机制

```
投入 → 用户数据/关系/内容积累 → 更强的触发 → 下次循环效果更强 → 内部触发形成
```

---

## Case Study: Instagram

**从摄影App到日常习惯**（2010-2018，日均使用53分钟）

- **触发**：朋友邀请通知（外部）→ 无聊时自动打开（内部）
- **行动**：无限滚动、下拉刷新模拟老虎机拉杆、单手操作
- **奖励**：不知道下一张照片是什么（猎物奖励）+ 点赞数不确定（社会奖励）
- **投入**：发布的照片、粉丝群、社交图谱——切换平台意味着全部丢失

**结果**：2个月达到100万用户，完全由习惯循环驱动。

---

## Usage Tips

1. **先确认产品值得成为习惯** — 低频需求不适合
2. **内部触发从用户情绪出发** — 找到产品能解除的那个负面情绪
3. **奖励必须有变化** — 固定奖励不会创造渴望
4. **投入阶段最易被忽视** — 让用户"存"东西在你的产品里
5. **测量习惯形成** — 追踪D7/D30留存和使用频率分布

```markdown
# Hook Model (Nir Eyal)

Build habitual engagement through a four-phase loop: Trigger, Action, Variable Reward, and Investment.

## Steps

1. **Trigger** -- Identify external triggers (notifications, emails) and the internal trigger (emotion/need) you want to own
2. **Action** -- Define the simplest behavior the user takes in anticipation of reward (Fogg: motivation + ability + trigger)
3. **Variable Reward** -- Design rewards with variability: tribe (social), hunt (resources), or self (mastery)
4. **Investment** -- Identify what the user puts in (data, content, reputation) that improves the next loop
5. Map how the investment creates or loads the next trigger

## Output Format

| Phase | Design |
|-------|--------|
| Internal Trigger | [Emotion or need] |
| External Trigger | [Channel and message] |
| Action | [Simplest user behavior] |
| Variable Reward | [Type: tribe/hunt/self + description] |
| Investment | [What user stores/creates] |

**Loop reinforcement:** [How investment loads next trigger]
```
