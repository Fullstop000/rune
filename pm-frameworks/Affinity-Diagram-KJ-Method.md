# Affinity Diagram / KJ Method

**Category:** Discovery (用户洞察)

## Description

Write qualitative data on sticky notes, cluster and group silently to discover patterns and insights from chaotic information.

## Process

1. **Record** - Write each idea/insight on a separate sticky note
2. **Sort** - Silently group related items together
3. **Label** - Create headers for each group
4. **Refine** - Iterate and adjust groupings
5. **Synthesize** - Draw conclusions from clusters

## When to Use

- Organizing large amounts of qualitative data
- Grouping user research insights
- Brainstorming session synthesis
- Finding patterns in complex information

## Skill Content

```markdown
# 亲和图 (KJ法)

混乱不是问题，混乱是原材料。把一切写下来，贴上去，然后让规律自己浮现。

## 基本信息

**提出者**: 川喜田二郎（Jiro Kawakita）
**来源**: 1960年代人类学田野调查方法
**适合阶段**: 定性研究整理 · 团队洞察对齐
**使用时长**: 小组练习 2–3 小时 · 大规模 半天–全天

## 01 它解决什么问题

亲和图最适合在大量定性数据面前「不知从哪里下手」的时刻。它是一种集体智慧的提炼工具，通过分组聚类让隐藏在混乱信息中的规律自然浮现。

**适用场景**:
- 用户访谈数据整理
- 头脑风暴后归纳
- 团队共识建立
- 客服反馈分类
- 产品问题聚类
- 跨部门知识整合

## 02 框架结构

一块白板，无数便利贴。亲和图的物质形态极其简单，但背后是一套严格的认知过程——先发散、后归类、再命名。

**三个阶段**:

1. **发散**: 每条写便利贴，一条数据一张贴，不评判不筛选，越多越好，保持沉默独立进行
2. **聚类**: 无声移动便利贴，靠直觉将相似的贴放在一起，可以打破他人的归类
3. **命名**: 为每组命名讨论，聚类稳定后为每个组写代表性标题

## 03 来源与历史

- **1960s**: 川喜田二郎田野调查方法
- **1967**: KJ 法正式命名出版
- **1986**: 引入软件工程与质量管理领域
- **2000s**: 成为 UX 研究标配工具
- **2020**: Miro 等工具数字化亲和图

## 04 真实案例

**IDEO × Kaiser Permanente — 急诊室体验重设计**

研究团队在4家医院收集超过800条患者、护士、医生的原始观察记录，通过亲和图法聚类出四大主题：

- **信息不透明**: 患者不知道排队位置，家属无法获知进展，交接班信息遗漏
- **流程低效**: 多系统重复录入，患者反复回答相同问题，医生护士术语不统一
- **物理环境障碍**: 护士每班走5英里找设备，推车堵塞走廊
- **情感忽视**: 患者感觉被遗忘，家属焦虑无处安置

**实施结果**: 患者满意度评分提升18%，护士每班步行距离减少29%，入院信息采集时间从40分钟降至17分钟。

## 05 使用步骤

1. **收集原始数据**: 整理访谈记录、用户反馈、观察笔记等
2. **每条写便利贴**: 每张只写一条，全组成员独立完成
3. **静默分组聚类**: 全程保持沉默，靠直觉移动相似的贴
4. **为每组命名**: 共同讨论，写下代表性标题
5. **输出核心洞察**: 将命名后的各组转化为洞察陈述

**✓ 这样做**:
- 严格执行「沉默聚类」规则
- 允许便利贴同时属于多个组
- 保留「未分类」区域
- 拍照保存每个阶段

**✗ 避免这些**:
- 不要在聚类前预设分类框架
- 不要让一个人单独完成所有便利贴
- 不要在命名阶段匆忙
- 不要只做一层聚类

## 配合使用

- **上游输入**: 情境调查法、用户访谈五问法
- **下游承接**: 同理心地图、HMW问题框架、机会解决方案树
```
