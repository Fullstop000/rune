# Contextual Inquiry

**Category:** Discovery (用户洞察)

## Description

Goes into users' real work/life contexts, observing while interviewing to capture natural behaviors impossible to observe in labs.

## Principles

1. **Context** - Observe users in their natural environment
2. **Partnership** - Learn WITH users, not just about them
3. **Interpretation** - Build hypotheses during observation
4. **Focus** - Let users guide the session while maintaining objectives

## When to Use

- Understanding natural user behavior
- Discovering workarounds and pain points
- Capturing context that users can't articulate
- Validating assumptions about user workflows

## Skill Content

```markdown
# 情境调研 Contextual Inquiry

## 什么是情境调研

走进用户的世界，而不是把他们带到会议室。真实场景里发生的，才是设计真正的原材料。

**核心隐喻**：师徒关系 — 用户是师傅，研究员是学徒。学徒在师傅的工作现场学习，而不是把师傅请到教室。

## 四大原则

- **情境 (Context)** — 在用户真实工作或生活环境中进行，不在实验室
- **伙伴关系 (Partnership)** — 研究员与用户共同探索，不是单向观察
- **诠释 (Interpretation)** — 解读当场与用户确认，避免私自假设
- **聚焦 (Focus)** — 带着研究目标进入，有意识引导调研深度

## 四大数据模型

1. **顺序模型** — 记录任务步骤顺序与触发条件
2. **文化模型** — 揭示组织规范、价值观与隐性规则
3. **流程模型** — 描述信息、工件和责任如何流转
4. **物理模型** — 记录空间布局与工具摆放

## 五步操作流程

1. **选择典型用户** — 选取 8-12 名代表性用户，涵盖核心场景
2. **进入真实场景** — 前往用户环境，说明目的建立信任
3. **观察与同步访谈** — "边做边说"，追问"你能演示给我看吗"
4. **记录关键行为** — 笔记、录音（需授权）、工作工件、环境细节
5. **提炼行为洞察** — 整合数据，绘制模型，提炼设计机会

## 适用场景

- 复杂工作流理解
- B2B 企业软件
- 医疗健康产品
- 儿童/老人产品
- 硬件与环境设计
- 跨文化产品设计

## 这样做 vs 避免这些

✓ **做**：
- 带着好奇心进入，不带预设
- 追问"你能演示给我看吗"
- 记录工作工件（便签、打印件）
- 调研后 24 小时内完成笔记整理

✗ **避免**：
- 不要打断用户工作节奏
- 不要问"你觉得理想功能是什么"
- 不要只在顺利时观察，故障和变通方案才是金矿
- 不要独自调研，双人组合效果更好

## 经典案例

**Microsoft Office 2003 可用性危机研究 (2003-2004)**

6 个月情境调研，238 名真实用户。发现：90% 用户只使用 5% 功能，功能寻找失败超 4000 次，帮助文档使用率仅 3%。

**设计影响**：驱动 Office 2007 Ribbon 界面革命，功能发现率提升 3 倍。
```
