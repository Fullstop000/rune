# Value Chain Analysis (价值链分析)

**Category:** Systems Thinking (系统思维)

**Framework ID:** #83

**Source:** https://pmframe.works/framework-value-chain-analysis

---

## Overview

Michael Porter: decomposes enterprise activities into primary and support activities to identify key links creating competitive advantage.

**Creator:** Michael Porter (Harvard Business School)

**Year:** 1985

---

## Primary Activities

| Activity | Description |
|----------|-------------|
| **Inbound Logistics** | Receiving, storing |
| **Operations** | Transforming inputs |
| **Outbound Logistics** | Distributing |
| **Marketing & Sales** | Selling |
| **Service** | Post-sale support |

## Support Activities

| Activity | Description |
|----------|-------------|
| **Procurement** | Sourcing |
| **Technology Development** | R&D |
| **Human Resources** | Talent |
| **Firm Infrastructure** | Management, finance |

---

## Case Study: Apple Inc. (2007-present)

- Operations: A-series/M-series chips enable deep hardware-software integration
- Outbound Logistics: Apple Stores control purchase experience
- Service: iCloud, AirDrop, Handoff create device ecosystem lock-in

# Value Chain Analysis (Porter)

Decompose the organization into strategically relevant activities to find where value is created or lost.

## Steps

1. Map **Primary Activities**: Inbound Logistics, Operations, Outbound Logistics, Marketing & Sales, Service.
2. Map **Support Activities**: Firm Infrastructure, HR Management, Technology Development, Procurement.
3. For each activity, assess cost drivers, value contribution, and current performance level.
4. Identify linkages — where one activity's performance affects another's cost or effectiveness.
5. Pinpoint activities that are sources of differentiation or cost advantage, and flag underperforming areas.

## Output Format

Value chain diagram with all activities labeled, cost and value assessment per activity, linkage analysis, and strategic recommendations for strengthening competitive advantage.
