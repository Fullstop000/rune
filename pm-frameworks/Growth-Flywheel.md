# Growth Flywheel (增长飞轮)

**Category:** Growth (增长策略)

**Framework ID:** #60

**Source:** https://pmframe.works/framework-growth-flywheel

---

## Overview

A strategic framework that designs core user behaviors as self-reinforcing closed loops. The central principle: "The more users engage, the more valuable the product becomes, attracting additional users and creating compound growth."

---

## Methodology

The flywheel consists of 4–6 interconnected nodes where each node's growth drives the next, eventually returning to the starting point. The critical insight is identifying the "rate-limiting node" — whichever node currently moves most slowly — as this determines overall flywheel speed.

---

## Key Components

| Aspect | Description |
|--------|-------------|
| **Structure** | 4-6 nodes per flywheel |
| **Requirement** | Each node must be quantifiable and optimizable |
| **Mechanism** | Closed loop: last node reinforces the first |

**Core Question:** "Which variable accelerates the entire system's rotation?"

---

## Steps

1. Map 4-6 core product behaviors as nodes
2. Validate each "A → B" relationship with data
3. Identify the current rate-limiting node
4. Concentrate resources on that bottleneck
5. Align quarterly OKRs to flywheel nodes
6. Repeat verification each cycle

---

## Classic Examples

**Amazon Flywheel:** Lower Prices → More Customers → More Traffic → More Sellers → More Selection → Better Experience → (cost reduction loops back to Lower Prices)

**Airbnb Flywheel:** More Hosts → More Listings → More Travelers → Reviews → Trust + Marketing → (loops back to More Hosts)

---

## Common Mistakes to Avoid

- Calling any circular diagram a flywheel
- Including more than 6 nodes
- Ignoring rate-limiting node identification
- Building on paid acquisition rather than product value
- Skipping causal relationship verification
- Using flywheel only for presentations, not decision-making

## Skill Content

# Growth Flywheel

Map the virtuous cycles where each step in the loop accelerates the next, creating compounding growth.

## Steps

1. Identify the **core action** users take that creates value
2. Map what that action produces as an **output** (content, data, network, etc.)
3. Trace how the output **attracts** more users or increases engagement
4. Identify what makes returning users **do more** of the core action
5. Draw the full loop and label the **force multiplier** at each connection
6. Find friction points that slow the flywheel and design interventions

[Action] -> [Output] -> [Attraction] -> [More Users] -> [Action] (loop)

**Force Multipliers:**
- [Connection 1]: [What amplifies it]
- [Connection 2]: [What amplifies it]

**Friction Points:** [Where the loop breaks down]
**Acceleration Plan:** [Top interventions to spin faster]
