# PURE Usability Evaluation (PURE 可用性评估)

**Category:** Validation (验证测试)

**Framework ID:** #44

**Source:** https://pmframe.works/framework-pure-usability-evaluation

---

## Overview

Expert walkthrough evaluating product experience quality across four dimensions—Purposeful, Usable, Reliable, Enjoyable—without users.

---

## Four Dimensions

| Dimension | Description |
|-----------|-------------|
| **Purposeful** | Does it serve user needs? |
| **Usable** | Can users easily accomplish tasks? |
| **Reliable** | Does it work consistently? |
| **Enjoyable** | Is the experience pleasant? |

---

## Process

1. **Select evaluators** — 3-5 usability experts
2. **Define tasks** — Representative user tasks
3. **Conduct walkthrough** — Experts evaluate each dimension
4. **Aggregate findings** — Compile issues
5. **Prioritize** — Address most critical problems

---

## When to Use

- Quick usability evaluation
- Before user testing
- Internal design review
- Heuristic evaluation

# PURE Usability Evaluation

Assess a product's user experience through four complementary heuristic lenses.

## Steps

1. **Purposeful** — does the product clearly serve a defined user need? Can users immediately understand what it does and why?
2. **Usable** — can users accomplish tasks efficiently? Check navigation, learnability, error recovery, and accessibility.
3. **Reliable** — does it perform consistently? Assess uptime, load times, data integrity, and edge-case handling.
4. **Enjoyable** — is the experience pleasant? Evaluate aesthetics, micro-interactions, tone, and emotional response.
5. Synthesize findings into a prioritized list of improvements ranked by user impact.

## Output Format

PURE scorecard rating each dimension (1-5) with specific findings, evidence, and a prioritized improvement backlog.
