# User Interview Five Questions

**Category:** Discovery (用户洞察)

## Description

Structured deep interviews progressing from behavior to motivation to psychology, excavating true needs and unmet desires.

## The Five Questions Framework

1. **Behavioral Questions** - What do you actually do?
2. **Motivational Questions** - Why do you do it that way?
3. **Psychological Questions** - What are your underlying needs?
4. **Contextual Questions** - What is the situation?
5. ** aspirational Questions** - What would you ideally want?

## When to Use

- User research and discovery
- Understanding user motivations
- Uncovering latent needs
- Building user personas

- **Suitable Phase:** 需求发现期 · 假设验证期
- **Duration:** 单次访谈 45–90 分钟 · 系列访谈 1–2 周
- **Origin:** 综合 UX 研究实践，2000 年代用户研究领域形成

## Core Framework

### 五层追问结构

| 层级 | 类型 | 核心问题 |
|------|------|----------|
| Q1 | 了解用户背景 | 「能跟我聊聊您平时工作中...是怎样的吗？」 |
| Q2 | 描述具体行为 | 「上一次您需要...时，具体是怎么做的？」 |
| Q3 | 探索行动动机 | 「为什么这对您来说很重要？」 |
| Q4 | 聚焦痛苦来源 | 「这个过程中什么地方最让您头疼？」 |
| Q5 | 描绘理想状态 | 「如果这个问题被完美解决了，会是什么样的？」 |

---

## Usage Steps

1. **了解用户背景** — 用开放问题打开对话，建立信任
2. **描述具体行为** — 要求描述「最近一次」，具体才有价值
3. **探索行动动机** — 反复追问「为什么」，至少三层
4. **聚焦痛苦来源** — 引导说出情绪化的真实感受
5. **描绘理想状态** — 让用户描述而非评价

---

## Best Practices

### ✓ 这样做
- 追问而非引导：「然后呢？为什么？能说得更具体些吗？」
- 沉默是金：用户沉默时等 5 秒，往往会说出更真实的话
- 问行为而非意见：「你上次怎么做的？」而非「你会怎么做？」
- 录音并逐字转录

### ✗ 避免这些
- 问「你会用这个功能吗？」用户几乎总是说「会」
- 一次问多个问题
- 提前透露产品方案
- 在 Q1-Q2 阶段就急于追问动机

## Applicable Scenarios

- 🌱 产品立项前（确保方向基于真实用户需求）
- 📊 功能上线后评估（补充数据指标无法反映的质性洞察）
- 👋 用户流失访谈（了解离开的真实原因）
- 🔀 竞品切换访谈（发现竞争格局中的真正优劣势）
- 🌍 国际化用户调研（理解当地用户行为背景和文化差异）
- 🔧 服务失败分析（还原用户真实经历）

## Key Insight

> 「最有价值的洞察通常出现在 Q4 和 Q5 之间：当用户描述完最痛苦的问题后，他们对『理想状态』的描述会格外真实和具体，往往直接对应着产品最该解决的核心任务。」
