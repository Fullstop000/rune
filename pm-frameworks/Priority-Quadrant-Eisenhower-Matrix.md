# Priority Quadrant / Eisenhower Matrix

**Category:** Execution (执行落地)

## Description

Places requirements by Importance and Urgency, distinguishing immediate action, scheduled, delegated, and eliminated items.

## The Four Quadrants

| | **Urgent** | **Not Urgent** |
|---|---|---|
| **Important** | **Do First** | **Schedule** |
| **Not Important** | **Delegate** | **Eliminate** |

## Actions

1. **Do First** - Important + Urgent = Do immediately
2. **Schedule** - Important + Not Urgent = Plan for later
3. **Delegate** - Not Important + Urgent = Have others do
4. **Eliminate** - Not Important + Not Urgent = Remove

## When to Use

- Personal productivity
- Task prioritization
- Time management
- Resource allocation

## Skill Content

```markdown
# 需求优先级四象限

## 概念

将需求按「重要性」与「紧迫性」两个独立维度放入四象限：
- **Q1 立即做**：重要 + 紧急
- **Q2 计划做**：重要 + 不紧急
- **Q3 委托做**：不重要 + 紧急
- **Q4 不做**：不重要 + 不紧急

核心洞察：「紧急」和「重要」是两个独立维度，最高价值工作往往是重要但不紧急的。

## 适用场景

- Backlog 梳理会
- 跨团队需求对齐
- 突发事件响应
- 季度路线图规划
- 个人时间管理
- 拒绝需求时提供依据

## 执行步骤

1. **对齐重要性标准**：先与团队对齐当前阶段核心目标，重要性以对核心目标的贡献度衡量
2. **客观评估紧迫性**：基于具体截止时间和成本，而非情绪焦虑
3. **物理化放置**：用白板画四象限，便利贴放置需求，集体讨论争议项
4. **为Q2主动排期**：预留 20%-30% 资源配额，不要等"有时间再做"
5. **定期回顾**：每 1-2 周动态调整分布

## 常见陷阱

- 「重要性」被职级而非目标定义
- Q1 永远满（慢性紧急状态）
- Q2 从不被执行
- Q4 不解释原因

## 配合工具

- RICE 优先级模型（量化排序）
- MoSCoW 方法（必要性分类）
- ICE 评分模型（影响力与信心度）
- Impact Effort 矩阵（精细权衡）
```
